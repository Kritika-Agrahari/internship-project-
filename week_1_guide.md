## 📋 Week 1 Checklist

Ensure you have completed all of the following before submitting:
- [ ] **Public Repository:** GitHub repository created and set to public.
- [ ] **README.md:** Filled out with project title, 1-line description, problem statement code, segment, name, and "What I learned" section.
- [ ] **Design Document:** Final version of your 1-page design doc saved in the repo under `docs/design_doc.md`.
- [ ] **Tech Stack Table:** Pre-defined components and choice justifications included.
- [ ] **Data Layer Working:** Ingested and queried data, verified with a screenshot or terminal log.
- [ ] **Git Commits:** At least **5 commits** pushed to the `main` branch.
- [ ] **Learning Reflection:** 3–5 concrete bullet points on technical concepts learned.
- [ ] **Status Update:** Completed "What's done", "What's stuck", "Goals for next week", and "Mentor help request".
