# ADR-001: Champion Model Choice (LightGBM)

## Context
We need to select a primary machine learning model for granular item-level demand forecasting. The dataset contains 5.8 million transactional records across 50,000+ store-item combinations. The model must handle non-linear patterns (such as promotions and pricing elasticity), leverage high-cardinality categorical features (Store IDs, Product Categories), and train in-memory within reasonable time limits on consumer hardware.

## Decision
We chose **LightGBM** (Light Gradient Boosting Machine) developed by Microsoft as the champion model, over XGBoost and classical SARIMA time-series models.

## Consequences
- **Training Speed**: LightGBM uses histogram-based split finding, reducing computational complexity from O(data) to O(bins). This allows the model to train on 5.8M rows in minutes compared to hours for XGBoost.
- **Accuracy**: Achieved an $R^2$ of 0.9228 and RMSE of 7.6788, outperforming XGBoost by 25% and SARIMA by over 50%.
- **Memory Footprint**: Leaf-wise tree growth produces deeper, more accurate trees with lower memory usage than level-wise algorithms.
- **Categorical Handling**: Native support for integer-encoded categories avoids one-hot encoding, preventing high-dimensional sparse matrices.
- **Operational Risk**: Deeper trees are prone to overfitting on small slices. Regularization (`min_child_samples`, `num_leaves`) must be tuned carefully.

## Alternatives Considered
1. **XGBoost**: Strong baseline, but slower training speed and lacks native categorical support (requires one-hot encoding, which consumes too much memory for high-cardinality categorical features).
2. **SARIMA**: High performance for aggregate trends, but impossible to scale. Training a local model for each of the 50,000+ store-item combinations is computationally infeasible and cannot capture cross-series interactions.
